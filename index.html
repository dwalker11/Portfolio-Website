What do I do?

Because writing code is only one phase of creating great software

Analyze

A consultation period necessary to gather requirements, research the problem domain, and transform a vague understand of what needs to be done into concrete tasks and solutions to be carried out by the software application.

Design

Translate programming tasks, defined in the analysis phase, into smaller components and create a high-level design of the application's architecture, which documents the relationship between each component of the software application.

Implement

Where actual coding takes place. This phase concerns itself with translating abstract design concepts into real executable code. It's where the rubber meets the road.

Test

The creation of various test that proactively finds underlying errors and verifies the correctness of the existing code. Usually accomplished by utilizing a testing driven framework or methodology.
